## 1. Java 21 虚拟线程

**现状**：Java 17 + Spring Boot 3.2.0

**学习目标**：
- 理解虚拟线程（Virtual Threads）与传统线程模型的区别
- 掌握虚拟线程的适用场景（IO密集型 vs CPU密集型）
- 在项目中启用虚拟线程

**实施要点**：
- 升级 JDK 到 21
- Spring Boot 3.2 已支持虚拟线程，配置方式：
  ```yaml
  spring:
    threads:
      virtual:
        enabled: true
  ```
- 替换 `CompletableFuture.supplyAsync()` 等异步操作为虚拟线程
- 注意：虚拟线程不适合 synchronized 块和 CPU 密集型任务

**参考资源**：
- JEP 444: Virtual Threads
- Spring Boot 3.2 Virtual Threads 支持文档

## 2. Docker 容器化

**现状**：项目无 Dockerfile，只能本地手动部署

**学习目标**：
- 理解 Docker 镜像分层原理
- 掌握 Dockerfile 编写
- 实现项目的多阶段构建

**实施要点**：
- 编写 `Dockerfile` 实现多阶段构建：
  - 阶段1：Maven 编译打包
  - 阶段2：JRE 运行时
- 使用 `spring-boot-maven-plugin` 生成分层 JAR，利用 Docker 缓存
- 配置 `.dockerignore` 排除无用文件

```dockerfile
# 示例结构
FROM maven:3.9-eclipse-temurin-21 AS build
WORKDIR /app
COPY pom.xml .
COPY common/pom.xml common/
COPY server/pom.xml server/
RUN mvn dependency:go-offline
COPY . .
RUN mvn clean package -DskipTests

FROM eclipse-temurin:21-jre
WORKDIR /app
COPY --from=build /app/server/target/*.jar app.jar
EXPOSE 8081
ENTRYPOINT ["java", "jar", "app.jar"]
```

**参考资源**：
- Docker 官方文档
- Spring Boot Docker 分层 JAR 文档

## 3. Docker Compose 编排

**现状**：MySQL、Redis、Nacos 均需手动启动

**学习目标**：
- 理解 Docker Compose 服务编排
- 掌握依赖启动顺序控制
- 统一管理项目所有中间件

**实施要点**：
- 编写 `docker-compose.yml` 编排以下服务：
  - `mysql` — 数据库
  - `redis` — 缓存
  - `nacos` — 注册中心/配置中心
  - `app` — 应用服务（依赖以上三者）
- 配置健康检查（healthcheck）确保启动顺序
- 使用环境变量管理配置，实现一键启动

```yaml
# 示例结构
services:
  mysql:
    image: mysql:8
    environment:
      MYSQL_ROOT_PASSWORD: 123456
      MYSQL_DATABASE: nacon7_db
    ports:
      - "3306:3306"
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]

  redis:
    image: redis:7
    ports:
      - "6379:6379"

  nacos:
    image: nacos/nacos-server:2.2.3
    environment:
      MODE: standalone
    ports:
      - "8848:8848"

  app:
    build: .
    ports:
      - "8081:8081"
    environment:
      DB_HOST: mysql
      REDIS_HOST: redis
      NACOS_SERVER_ADDR: nacos:8848
    depends_on:
      mysql: { condition: service_healthy }
      redis: { condition: service_started }
      nacos: { condition: service_started }
```

**参考资源**：
- Docker Compose 官方文档
- Docker Compose healthcheck 用法

## 4. Kubernetes 部署

**现状**：无 K8s 清单文件

**学习目标**：
- 理解 Kubernetes 核心概念（Pod、Service、Deployment、ConfigMap）
- 掌握 K8s 资源清单编写
- 实现应用的声明式部署

**实施要点**：
- 编写以下 K8s 资源清单：
  - `deployment.yaml` — 应用部署（副本数、滚动更新策略）
  - `service.yaml` — 服务暴露（ClusterIP / NodePort）
  - `configmap.yaml` — 配置管理

**学习路径**：
1. 先学 minikube / kind 搭建本地集群
2. 掌握 kubectl 常用命令
3. 将项目部署到本地集群

**参考资源**：
- Kubernetes 官方教程

## 5. 消息队列

**现状**：无消息队列，同步调用，缺乏异步解耦能力

**学习目标**：
- 理解消息队列的应用场景（削峰填谷、异步解耦、最终一致性）
- 掌握至少一种 MQ（推荐 RabbitMQ 入门，Kafka 进阶）

**在项目中的落地场景**：
1. **借阅异步通知**：借书成功后发送消息 → 邮件/短信通知用户
2. **逾期检测**：定时任务扫描逾期记录 → 发送消息触发提醒
3. **操作日志**：关键操作（删除用户/图书）发送消息 → 异步落库审计日志

**实施要点（以 RabbitMQ 为例）**：
- 添加依赖 `spring-boot-starter-amqp`
- 定义交换机（Exchange）、队列（Queue）、绑定关系
- 使用 `@RabbitListener` 消费消息
- 配置消息确认机制（Publisher Confirm + Consumer Ack）

**参考资源**：
- RabbitMQ 官方教程
- Spring AMQP 文档
- Kafka 官方文档（进阶）

## 7. Actuator 全面监控

**现状**：当前只暴露了 health、info 端点

**学习目标**：
- 理解 Spring Boot Actuator 各端点用途
- 掌握生产级监控配置
- 集成 Micrometer + Prometheus 指标采集

**实施要点**：
- 开启更多端点：metrics、env、mappings、threaddump、heapdump
- 集成 `micrometer-registry-prometheus` 暴露 Prometheus 格式指标
- 集成 Spring Boot Admin 实现可视化监控面板
- 自定义健康检查（如：检测 Redis / Nacos 连接状态）

```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics,prometheus,env,mappings,threaddump
  metrics:
    export:
      prometheus:
        enabled: true
  endpoint:
    health:
      show-details: always
```

**参考资源**：
- Spring Boot Actuator 官方文档
- Micrometer 文档
- Prometheus + Grafana 入门

## 8. 分布式日志追踪

**现状**：无链路追踪能力，日志分散难以排查

**学习目标**：
- 理解分布式追踪原理（Trace ID + Span ID）
- 掌握 Micrometer Tracing 的使用

**实施要点**：
- 添加依赖 `io.micrometer:micrometer-tracing-bridge-brave`
- 配置日志格式自动携带 Trace ID
- 集成 Zipkin 实现链路可视化

```yaml
logging:
  pattern:
    level: "%5p [%X{traceId:-},%X{spanId:-}]"
```

**参考资源**：
- Micrometer Tracing 官方文档
- Zipkin 文档
